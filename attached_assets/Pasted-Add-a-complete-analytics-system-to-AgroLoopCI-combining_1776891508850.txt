## PART 1 — GOOGLE ANALYTICS 4

## Setup

Install Google Analytics 4 in the React app:

1. Add GA4 script to index.html <head>:
<!-- Google tag (gtag.js) -->
<script async 
  src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX">
</script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>

2. Store measurement ID in .env:
   VITE_GA_MEASUREMENT_ID=G-XXXXXXXXXX

3. Create analytics helper (src/lib/analytics.ts):

// Track page views (call on route change)
export const trackPageView = (path: string) => {
  window.gtag('config', import.meta.env.VITE_GA_MEASUREMENT_ID, {
    page_path: path
  })
}

// Track custom events
export const trackEvent = (
  action: string, 
  category: string, 
  label?: string, 
  value?: number
) => {
  window.gtag('event', action, {
    event_category: category,
    event_label: label,
    value: value
  })
}

4. Add to App.tsx — track route changes:
const location = useLocation()
useEffect(() => {
  trackPageView(location.pathname)
}, [location])

## PART 2 — CUSTOM ANALYTICS (admin dashboard)
═══════════════════════════════════════════

## DATABASE

New page_views table:
- id, path (text), referrer (text nullable)
- user_id (FK users, nullable — null if not logged in)
- session_id (text — random UUID per session)
- device_type (enum: 'mobile' | 'tablet' | 'desktop')
- region (text nullable — from user profile if logged in)
- duration_seconds (integer nullable)
- created_at

New offer_views table:
- id, offre_id (FK residus)
- viewer_id (FK users, nullable)
- session_id (text)
- created_at

New conversion_events table:
- id, event_type (enum: 
    'visit' | 'register' | 'first_offer' |
    'first_contact' | 'first_transaction' |
    'subscription')
- user_id (FK users, nullable)
- session_id (text)
- metadata (jsonb)
- created_at

## BACKEND

Analytics middleware (middleware/analytics.ts):
Applied to ALL routes automatically:

- Extract session_id from cookie 
  (set cookie if not exists: random UUID, 30min expiry)
- Detect device_type from User-Agent:
    mobile: /mobile/i
    tablet: /tablet|ipad/i
    desktop: default
- On each request:
    INSERT into page_views (non-blocking, async)
    Don't await — fire and forget

Specific tracking:

On GET /api/offres/:id:
  INSERT into offer_views
  INCREMENT residus.view_count

On POST /api/auth/register:
  INSERT conversion_events: 'register'

On POST /api/offres (first offer):
  Check if user has previous offers
  If none: INSERT conversion_events: 'first_offer'

On POST /api/conversations (first contact):
  Check if user has previous conversations
  If none: INSERT conversion_events: 'first_contact'

On PUT /api/transactions/:id (status→confirmée):
  INSERT conversion_events: 'first_transaction'
  (if first for this user)

On POST /api/subscriptions/subscribe:
  INSERT conversion_events: 'subscription'

GET /api/admin/analytics/traffic:
- Query params: period (7|30|90 days), granularity (day|week)
- Returns time series:
  [{ date, visitors, page_views, new_users }]

GET /api/admin/analytics/offers:
- Top viewed offers with conversion rate
- Most shared offer types
- Average time on offer page

GET /api/admin/analytics/conversions:
- Full funnel data
- Drop-off rates between steps
- Average time between steps

GET /api/admin/analytics/realtime:
- Active visitors RIGHT NOW (session in last 5 min)
- Pages they are on
- Refreshed every 30 seconds
