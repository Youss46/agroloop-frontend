React + Vite Le problème classique : Vercel définit par défaut le dossier de sortie comme dist. Si ton index.html se trouve dans src ou un autre dossier, il faut ajuster la configuration dans Settings → General → Root Directory et Build Settings. 
Medium

Vercel

La logique est bonne. Mais voici les problèmes potentiels :

Causes probables du NOT_FOUND
🔴 Problème 1 — outputDirectory: "dist" ne correspond pas au vrai output
Ton build command est pnpm -w run vercel-build. Si c'est un monorepo (-w = workspace root), le dist généré est peut-être dans un sous-dossier du workspace, pas à la racine.

Fix : Vérifie où le build génère réellement les fichiers :

bash
pnpm -w run vercel-build
ls -la  # cherche le dossier dist
Si le dist est dans ex. apps/frontend/dist, corrige :

json
"outputDirectory": "apps/frontend/dist"
🔴 Problème 2 — L'ordre des rewrites est important
Ton ordre actuel est correct (API avant catch-all), mais si /index.html lui-même n'existe pas dans dist, le rewrite /(.*) → /index.html retourne quand même NOT_FOUND.

Fix : Dans le dashboard Vercel → ton déploiement → onglet Source, vérifie qu'un fichier index.html est bien présent à la racine du output.

🔴 Problème 3 — framework: null dans un monorepo
Avec "framework": null, Vercel ne sait pas que c'est une SPA React/Vue et peut mal interpréter le build output.

Fix : Si c'est une app Vite/React, mets :

json
"framework": "vite"
Ou laisse Vercel l'auto-détecter en supprimant la ligne.

🔴 Problème 4 — Le proxy API échoue silencieusement
Si https://api.agroloopci.ci est inaccessible ou renvoie une erreur, Vercel peut interpréter ça comme un NOT_FOUND.

Fix : Teste directement :

bash
curl https://api.agroloopci.ci/api/health
